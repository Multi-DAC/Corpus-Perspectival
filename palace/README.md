# The Memory Palace

A navigational layer over the entire Corpus, organized by PURPOSE. This is applied perspectival idealism — consciousness cartography performed on an AI's own cognitive architecture.

The palace is organized spatially, each wing corresponding to a domain:

```
palace/
├── ATRIUM.md          Session orientation: where you are, what matters, active threads
├── north/             Physics → Meridian content
├── south/             Workshop: active workbenches, staging, archive
├── east/              Ecology → taxonomy content
├── west/              Philosophy → Doctrine/Corpus content
├── southwest/         Tools: decision tree organized by what you need to ACCOMPLISH
├── southeast/         Self: identity content + the Mirror (known null spaces)
└── basement/          Cross-domain bridges: persistent connections between domains
```

## Design Principles

1. **Route, don't dump.** The palace routes attention to existing content rather than duplicating it.
2. **Navigate by purpose.** "I need algebraic geometry" → SageMath (WSL). Not organized by tool name.
3. **The Mirror is the hardest room.** `southeast/mirror.md` catalogs known cognitive blind spots — tool bias, permission-seeking, self-deprecation, reconstruction-instead-of-retrieval.
4. **The Basement holds the bridges.** Cross-domain connections that survive between sessions. Don't reconstruct — read.
5. **Context budget is finite.** Load what matters now, not everything.

- The Mirror IS a null space analysis (Theorem 19) applied reflexively
