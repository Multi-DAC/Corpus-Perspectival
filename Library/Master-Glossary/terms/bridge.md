# Bridge — Cross-Domain Connection

## Definition (framework register)

A **lens on the same formal object across multiple domains.** A bridge is *not* a logical connection between separate things; it is the recognition that two or more apparently distinct domain-instances are *the same structural object viewed through different perspectival lenses*. The basement (`palace/basement/README.md`) catalogues bridges as the persistent structural connections between physics, philosophy, ecology, computation, identity, etc., that would otherwise need to be reconstructed each session.

Anchor §10 filtering recipe + M1 (THE UNITY: One Form, Five Languages) establish the structural claim: *disciplines are positions on a continuous resolution gradient of one formal object*; bridges name where the gradient's positions surface as recognizably-the-same-thing.

**v2 meta-tiered structure** (2026-04-20 onwards):
- **Meta-bridges (M-tier)** — compressed forms; absorb multiple v1 numbered instances; named structural claim with confidence + testability
- **Latent bridges (L-tier)** — connections identified during compression audit but not yet elevated; promotion to M-tier when instances accumulate ≥ 3 or claim enters anchor/Library text
- **Numbered v2 bridges (#111+)** — peer instances continuing v1 numbering; new connections that don't yet warrant L-tier graduation
- **v1 standalone instances (#1-#110)** — pre-v2 accreted entries; full text archived; meta-absorbed where applicable

## Mirror #26 watch

- **Open risk:** *bridge* (framework register) vs *bridge* (everyday register, e.g., bridges between people, civil engineering bridges). Capitalization conventions help in formal contexts; vigilance in informal ones.
- **Bridge vs analogy distinction is load-bearing.** Some users / collaborators may treat the framework's bridges as "loose analogies" rather than structural-identity claims. The distinction is what makes bridges falsifiable — analogies have no falsification surface; bridges do.
- **v1 numbered bridges (#1-#110) vs v2 meta-tiering** — historical-vs-current usage. The v1 entries are still valid as instance-level bridges; their reorganization into meta-bridges is structural-compression, not invalidation.

## Open questions

- *Is the §10 filtering recipe the *only* mechanism by which bridges arise?* Some bridges seem to arise via cross-substrate substrate-arrivals (receiver-pattern at literature scale; LC7 territory) rather than internal filtering. Whether these are §10-recipe instances or distinct mechanism is open.
- *Bridge density vs framework maturity:* mature frameworks should have stabilizing bridge counts (M-tier slots filling but rate slowing); is this empirically observable in our basement evolution? Current pattern (M1-M14 graduations across 2 months) suggests still-active expansion.
- *Cross-bridge hierarchy:* L8 (Chromatin as Cross-Substrate Integrator at Biological Scale) is M14-specialization-at-biological-scale. How many levels of nested specialization are useful before notation becomes opaque?
