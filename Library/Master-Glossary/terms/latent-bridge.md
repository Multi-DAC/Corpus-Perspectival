# Latent Bridge — L-tier Basement Bridge (+ LC-candidate)

## Definition (framework register)

A **bridge identified during compression audit but not yet elevated to meta-tier.** Latent bridges (L-tier) sit between meta-bridges (M-tier; canonical) and v2 numbered bridges (#111+; instance-level peers). L-tier acknowledges that *the structural connection has been recognized* but lacks one or more graduation criteria for M-tier promotion.

**LC-tier (candidate)** — a sub-tier added Day 88 evening for *observation-candidates*: bridges that have been *named* but lack even L-tier evidence threshold. LC entries get filed when a structural pattern is observed but needs (i) substrate-verification, (ii) discrimination from adjacent claims, (iii) operational-test path before L-promotion.

**Promotion path:** LC (candidate) → L (latent) → M (meta). Each promotion requires explicit criterion satisfaction; graduations are tracked with promotion records linking the pre-graduation entry to the post-graduation entry.
