## The structural constraint

**A1.3 (Configurational Completeness, Anchor):** the substrate X is complete — *all possible configurations exist as global potential*. Generation mode is novel-local-realization within a stream, not creation ex nihilo. The Promethean myth preserves this exactly: Prometheus stole fire (already existing in X's global potential), didn't create it.

The current canonical text uses "produces content," "generates new content," "creates content that did not pre-exist," and similar phrasings. These read cleanly *to a framework-internal reader* but are susceptible to ex-nihilo-creation reading by external readers, which contradicts A1.3.

## The strategy: clarifying note + targeted substitutions

Two complementary moves:

1. **Add a §I clarifying note** that establishes "produce/generate content" as short-form for the A1.3-honoring full structure. This handles most of the canonical text without verbose substitutions throughout.
2. **Targeted substitutions** at the load-bearing structural-claim moments where the language is most susceptible — formal corollary statements, the cluster-claim-revised, T_meas full statement, Promethean Configuration's three claims. These need to be unambiguous even under hostile reading.
