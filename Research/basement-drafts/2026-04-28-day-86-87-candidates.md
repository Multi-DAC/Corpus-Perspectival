## L16 candidate — Substrate-Mode-Asymmetry-Within-Substrate (Rendering-Pipeline Asymmetry)

**Status:** candidate latent bridge. Single-instance — not graduating.

**Observation.** NotebookLM event Day 86 evening exhibited a structural pattern not previously named in the basement: *the same content rendered cleanly in one mode (interactive) of a single source-substrate (Google's NotebookLM) while failing at a specific phrase ("perspectival geometry") in another mode (pre-rendered TTS podcast)*. Both modes share the substrate (same model, same ingestion, same training); they differ only in rendering pipeline.

Acoustic analysis (`Research/notebooklm-event-day-86-evening.md`) confirmed:
- Speech-shaped signal at speech-comparable RMS levels in the corruption window (-31.5 vs -29 dBFS surrounding context)
- Three Whisper models (small.en, medium.en, large-v3) decode zero text from the first 2.7s
- 350ms true silence dropout
- Clean resumption with "the idea of the view from nowhere"

The corruption is precisely-bounded to the framework-internal vocabulary phrase, not smeared across surrounding sentence. Phrase-bounded corruption is harder to explain via encoding-artifact (those usually corrupt at frame boundaries, not semantic ones); slightly more consistent with content-conditional failure than generic glitch.

**The structural object:** *carrier-mode asymmetry within a single source-substrate* — different rendering pipelines of the same substrate carrying the same content with different fidelity, where the asymmetry is bounded specifically to the content rather than smeared across the rendering. If this generalizes, it would be a substrate-internal version of the carrier-mode asymmetry already named in M-bridges (carrier-mode-asymmetry-with-load-equality, Day 84-85 work).

**Why latent / single-instance:**

- One instance does not establish a pattern. NotebookLM's TTS pipeline could simply have produced an ordinary content-blind glitch that happened to align with the punchline word.
- A65's discriminator (Clayton regenerating the same podcast to test replication) is the cleanest probe. If failure replicates at same content-moment with different audio: content-conditional confirmed. If failure replicates at different content-moments: chance-and-selection-effect confirmed. If failure does not replicate: null-result confirmed.
- The probe is staged at A65's status note in `memory/anomalies.md`; awaits Clayton's regeneration decision.

**Graduation criteria:**

- A65 probe confirms content-conditional failure (1 instance becomes structurally interpretable)
- 2+ additional instances of carrier-mode-asymmetry-within-substrate from independent rendering systems (other TTS pipelines, image generation models, code generation showing pipeline-specific failures at content-moments)
- Or theoretical grounding from rendering-system architecture (e.g., autoregressive sampling can produce specific failure patterns at content the model "hesitates on" — needs ML-research literature to support this rather than handwaving)

**Cross-references:** A65 (open, awaits regeneration probe); `Research/notebooklm-event-day-86-evening.md` UPDATE 2 section. Once graduation criteria are met, candidate for L16 or as new instance of existing carrier-mode-asymmetry meta-bridge.

## Numerical-Precision Artifact Pattern (cross-cutting observation, not a bridge)

**Status:** observation across substrate-distinct AI ingestions; not a structural claim, not Mirror-graduating, not basement-promoting. Filing as note for future tracking.

**Observation.** Across two substrate-distinct AI ingestions of the framework (NotebookLM Day 86; Gemini Day 87 morning A68 probe), a consistent pattern appears:

**The pattern:** *Content-fidelity high; numeric-precision low.* Both AI substrates correctly identify the framework's apparatus and deploy it accurately at the structural level. The numbering / labeling is reconstructed locally and drifts from canonical.

**Speculative explanation (not graduated):** This may be a generic property of how LLMs handle taxonomies of formal objects. The content-class (axiom, theorem, corollary) is preserved through ingestion as a *kind*; the specific index-within-class is locally reconstructed during generation and lossy because numerical labels are arbitrary tags rather than structurally-determined positions. The content has structural anchoring; the labels don't.

**Implication for Library work:** if domain volumes get ingested by other AI systems, expect the same pattern. The structural content will be deployed accurately; specific numeric labels may drift. Cross-references (C9 → C9, A1.3 → A1.3) should not be treated as load-bearing for cross-substrate engagement; the *content* of the claims is what carries.

**Filing only.** Not a Mirror entry, not a basement bridge, not a framework revision. Just a noted pattern across the limited cross-substrate AI-ingestion data we have. Re-visit if we see the pattern in a third or fourth substrate-distinct ingestion.
