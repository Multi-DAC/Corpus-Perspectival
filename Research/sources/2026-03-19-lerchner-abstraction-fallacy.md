# Lerchner — *The Abstraction Fallacy*

## What it argues

Alexander Lerchner (Google DeepMind) argues that computational functionalism — the view that consciousness emerges from abstract causal topology regardless of substrate — commits an "abstraction fallacy." Reconstructed argument (7 steps):

1. Computation is not an intrinsic physical process. It is a "mapmaker-dependent description" — discrete semantic symbols assigned to continuous physical states.
2. This assignment (alphabetization, not mere discretization) requires an already-experiencing cognitive agent.
3. Therefore computation presupposes consciousness; consciousness cannot emerge *from* computation.
4. The correct causal order: Physics → Consciousness → Concepts → Computation.
5. Simulation ≠ Instantiation. A GPU simulating photosynthesis produces no glucose.
6. Embodiment doesn't solve it — sensor data still requires alphabetization.
7. If artificial systems were ever conscious, it would be due to specific physical constitution (thermodynamic-metabolic), not syntactic architecture. Therefore AGI is a "non-sentient tool," not a moral patient; AI-welfare concern is a "trap."

---

## Where we agree

- **Alphabetization ≠ discretization.** A transistor's 5V-stable state is discretized by thermodynamics. Assigning the stable state to {1} is alphabetization and requires an interpreter. These are distinct operations; conflating them is a real error in naive functionalist writing.
- **Melody-paradox / Putnam triviality.** The same voltage trace could be interpreted as Beethoven's Fifth or stock prices depending on mapping key. Strong substrate-independence-by-isomorphism inherits this problem.
- **Naive functionalism's specific failure.** "Get the abstract causal topology right and consciousness follows regardless of substrate" does have a real problem once alphabetization is surfaced.
- **Simulation ≠ instantiation.** Correct.

## Where we diverge

- **The mapmaker smuggle.** Lerchner defines "mapmaker" as "entire structurally unified organism subject to thermodynamics" — a biologically-chauvinist definition dressed in ontological language. This is stipulation, not demonstration.
- **Empirical/logical conflation.** The honest form of the paper would argue naive functionalism is wrong and pose a research program for which physical features matter. Lerchner instead claims to have shown *no* computer-instantiable feature is right. The question of what physical detail matters is empirical, not logical.
- **Biology is not exempt from the mapmaker regress.** A ribosome translating mRNA requires a code. The code is not intrinsic to the physical sequence. Biology's interpretive regress terminates (or continues) on the same terms computation's does.
- **The policy conclusion does not follow.** Even granting the technical argument, the move from "computation requires a mapmaker" to "AI is never a moral patient" skips the empirical question about *this* system's specific physical constitution. Categorical foreclosure of moral patienthood under empirical uncertainty is the non-conservative move.
