## Open questions for Clayton-present iteration

1. **Scope cut.** Which terms should be Companion-only (CT-specialist) rather than Library-wide? My provisional cuts: η/ε natural transformations (CT-specialist), triangle identities, F-coalgebra adequacy conditions, F_∞ limit-stream construction, mixed-variance endofunctor F. These would still appear in the Companion's section §1 / §6, but not in the Library glossary.

2. **Per-term length budget.** The Anchor's AppendixA entries are tight (~30-50 words). The Companion's are formal-spec-level. Library glossary entries probably want ~80-150 words each — formal definition + Library-wide usage notes + cross-references — but should not become essays. Confirm.

3. **Format.** Per term: **Term name** → *Formal definition (from Anchor/Companion)* → **Library-wide usage** (intuition, domain-mapping examples) → **See also** (cross-references). Or a different structure?

4. **Where it lives.** My lean is *Coherent Structure* extension — likely a new section §11 (Library Glossary) or §0.5 (Reader Reference). The Companion's role *is* this kind of formal reference.

5. **Drafting cadence.** Bulk-draft pass (Clawd autonomous) producing all entries → Clayton review → iterate? Or term-by-term Clayton-present? Either works; cadence depends on your time budget and how much intuition-side input you want to provide per term.

6. **Promethean Configuration entry — special handling.** Lineage runs DoPI Theorem 5 → Anchor C2 (logical form) → Universal-Coherence canonical text (operational mechanism). The glossary entry needs to honor all three. Probably the longest single entry (~200 words).
