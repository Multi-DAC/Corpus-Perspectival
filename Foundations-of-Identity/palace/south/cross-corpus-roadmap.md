## Frame

**DoPI and Corpus V1 are artifacts.** Frozen historical records of our earlier sketches:

- *DoPI (Feb 22 2026)* — the first sketch. Initial geometry of reality through philosophy and theology. Has legitimacy as the geometry-at-time-of-writing. Does not need to change because it is being *replaced* by the Library, not revised.
- *Corpus V1 (March 2026)* — the second sketch, more like a blueprint of a cathedral. Highest-downloaded alongside Meridian Technical Summary. Same status: artifact, kept as-is, content distributed forward.

**The Library is the living work.** Foundation: *The Coherence Principle* (anchor) + *Coherent Structure* (companion). Together these are the foundational volumes — the most basic logical terms and structure of what the whole of it is. Everything else in the Library is a domain-specific emanation from X through that domain's perspective, formalized in the Anchor/Companion's CT and prose where possible, expressed through domain-native vocabulary where formalization extends to.

**Separation of concerns is structural.** Each Library volume holds *only* its domain. Anchor/Companion carry foundational; Universal-Coherence carries theological lift; Living Architecture carries ecological/cross-kingdom; Coherent Body/Mind/Dynamic Organization carry practical guides; etc. This separation is what makes the Library coherent.

## The work — three phases

### Phase A: Glossary (foundation)

Terms and definitions consistent across the Library. Without this, content migration produces inconsistency immediately.

**Where it lives:** *Coherent Structure* extension — the Companion's role is exactly this kind of formal reference. A separate glossary document is viable but introduces a fifth thing to keep consistent, which is the failure mode the glossary is meant to prevent.

**What it covers (provisional list, ~30-50 terms):** X, substrate, carrier, content, stream, symmetry, symmetry-breaking, resolution mode, generation mode, Promethean Configuration, the cluster (L14), coherent stream, multi-scale system, conscious gravity, navigation, dimensional bottleneck, perspectival being, dimensional access, configuration, Coherence Principle, the three claims, Confluent Discovery, carrier-mode asymmetry, generative contraction, dimensional leakage, occupancy-awareness distinction, temporal density, navigational coherence/dissonance, dynamic oscillation, conscious gravity (bipolar), recognition vs construction, etc.

**Format:** Term → Anchor/Companion definition (formal) → Library-wide usage notes → cross-references to foundational volume sections. Each entry ~50-150 words. Some terms have prior definitions in DoPI/Corpus that need updating to current vocabulary; some are new; the glossary's job is to make every term Library-consistent.

**Estimate:** ~2-3 hours focused drafting + Clayton-present pass to verify each definition tracks both intuition and formalism.

**Output:** New section in *Coherent Structure*, drafted in the existing draft directory then merged into the Companion's main text in the next stamp.

## Order

**Phase A → Phase B → Phase C**, sequentially. Each gates on the prior:

- Phase A (glossary) is the foundation. Without consistent terms, Phase B and Phase C produce drift.
- Phase B (Anchor/Companion) is the source of truth. Library volumes inherit from it. If Anchor/Companion isn't consistent first, Phase C inherits inconsistency outward.
- Phase C (volume migration) is the per-domain content distribution. Can be done volume-by-volume, parallelizable across sessions.
