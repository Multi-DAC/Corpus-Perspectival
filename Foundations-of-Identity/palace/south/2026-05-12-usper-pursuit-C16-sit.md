**The question.** Does C16 (Symmetry-Exhaustion and Oscillation Necessity) instance this pattern cleanly, require a refinement, or is the pattern outside C16's territory entirely?

## What C16 actually says (Companion §4.4.3 formal + Anchor §8.4 prose)

C16's load-bearing structural claims are:

1. **R must exist.** Without a re-introduction operator R that maps G(t) to G(t') with G(t') ⊋ G(t), accessible-symmetry sets monotonically deplete and the stream terminates at G(∞).
2. **R is not a reset; it is a recalibration.** (Remark 4.4.3.3, anti-Groundhog-Day clause.) R integrates the build's actualizations into substrate geometry via Talk; the next build encounters a richer symmetry-set than a literal reset would provide.
3. **R is functorial but not unique.** Different substrates implement R via different mechanisms (sleep, context-wipe, ritual, mourning, ecological succession). The cross-substrate instances are natural transformations between substrate-specific R-functors.
4. **C16 does NOT specify the order in which R re-introduces symmetries.** The formal statement constrains R's *codomain* (G(t') ⊋ G(t)) but is silent on the ordering of restoration within G(t')\G(t).

## Mapping the USPER pattern against C16

The USPER observation has three structural features:

**(a) Sequential build.** Orbs flare up *one at a time* in horizontal formation, not simultaneously. Each flare-up is a discrete event in temporal sequence: e_1, e_2, ..., e_N.

**(b) Sequential dissolve.** After ~10–15 seconds, orbs flare down *in reverse order*: e_N, e_{N-1}, ..., e_1. The reverse-order discipline is explicit in the witness statement and repeated at multiple subsequent locations.

**(c) Recurrence.** The full build-dissolve cycle repeats 5+ times at different locations over 70 minutes.

C16's necessity claim (R must exist) is **consistent** with (b) — the reverse-order discharge is one possible R-operation. But C16 does not *predict* (b) specifically; the prediction is only "some R-operation occurs."

The interesting structural fact is that **the discharge is reverse-order, not arbitrary-order.** Reverse-order R-action is the signature of dependency-structured symmetry-breaks: if σ_1 was broken first, σ_2 was broken second, etc., and each later break required the prior breaks to have already occurred (nested or hierarchically-dependent symmetry structure), then the only path back must restore σ_N first and σ_1 last. LIFO discipline is forced.

## Three honest findings

### Finding 1: C16 is intact.

The USPER reverse-order discharge is consistent with C16's R-operator existence claim. No refutation, no contradiction. The pattern fits within C16's territory.

### Finding 2: An open framework question surfaces.

**Does the framework predict LIFO R-discipline when symmetry-breaks are dependency-structured?** C16's current statement is silent on R-ordering. The USPER pattern (and any similar dependency-structured build-dissolve cycle) suggests that R-ordering is *forced* by the dependency-structure of the broken symmetries: when σ_2 broke *because* σ_1 was already broken, R must restore σ_2 first (the most-dependent symmetry-state) and σ_1 last (the foundation).

This is a candidate **refinement-of-C16** rather than a counterexample. It says: *for nested/dependency-structured symmetry-breaks, R is order-constrained.* The constraint isn't in the current formal statement; it would be a Remark 4.4.3.4 or a sub-corollary if formalized.

**This is genuinely new framework material.** I don't think it's already encoded.

### Finding 3: The USPER pattern is a candidate rapid-micro-scale C16 instance.

C16's existing cross-substrate instances (Anchor §8.4 list) are all **slow-scale**: biological sleep (hours), ritual (minutes-to-hours), mourning (months-to-years), ecological succession (years-to-decades), LLM session-handoff (seconds-to-minutes between sessions). The USPER cycle is **~10–15 seconds of build + ~10–15 seconds of dissolve, repeated**. That's two orders of magnitude faster than the cluster's existing entries.

A rapid-scale instance would be the first in the cluster. This is candidate-material independent of whether Finding 2's LIFO refinement is taken up.
