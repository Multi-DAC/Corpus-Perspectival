## Attempt 1: Decomposability

A constraint lattice is DECOMPOSABLE if it can be written as a direct sum of independent sublattices: L = L₁ ⊕ L₂ ⊕ ... ⊕ Lₙ. Each sublattice operates independently — removing Lₖ doesn't affect L_j≠k.

The non-Abelian sector is INDECOMPOSABLE — removing one generator destabilizes the whole algebra (the structure constants link everything). This is why cascade failures happen: sediment one DOF, the coupling to adjacent DOFs creates stress, which sediments the next, etc.

**COMPRESS:** Robustness = decomposability. Fragility = indecomposability. The Abelian exception is a theorem about direct sums.

## Attempt 2: Information-Theoretic

A sedimented constraint is one that has been absorbed into the background — it no longer carries information because it's no longer free. Mutual information: I(Lₖ; Lⱼ) = 0 for Abelian DOFs. The sedimentation of Lⱼ transmits zero information to Lₖ.

**COMPRESS:** Sedimentation cascades via shared information. Abelian isolation = zero mutual information = cascade-proof.

## Attempt 3: Topological

No instantons = no tunneling path = the system stays in its vacuum. The Abelian sector can't tunnel because there's nowhere to tunnel TO.

**COMPRESS:** Sedimentation requires a topological pathway. π₃ = 0 means no pathway exists. Abelian immunity is topological impossibility of vacuum transition.
