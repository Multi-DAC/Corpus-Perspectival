**Two-part diagnosis:**

1. **Supervisor target was the load-bearing bug.** BCE(confidence_at_halt, per_batch_mean_correctness) pushes confidence *toward* per-batch correctness. When correctness ~0.65 (most of training after warm-up), the BCE supervisor *trains* the Mirror to assert confidence ~0.65 — which is above the 0.5 halt threshold. The supervisor is the mechanism that drives collapse. I designed the wrong target.

2. **Channel leakage via attention-pool backward gradient.** The pre-reg's "zero channel-modulation DOF" specified zero FORWARD-PASS modulation. But the supervisor loss flows backward through the MirrorMeasurer's attention pool, then into z_p and z_e gradients, then into channel parameter updates. The Mirror's read-only forward output doesn't influence channel dynamics, but its READING does — via the optimizer's joint gradient. This is the "hidden gradient pathway" the pre-reg §6 outcome table named as a failure-mode candidate.

**What Read B is NOT yet falsified by:** v3h's implementation cannot cleanly test Read B because the implementation has channel leakage AND the supervisor target was structurally wrong. A clean re-test requires:
- (a) `z_p.detach()` and `z_e.detach()` at the Mirror's input boundary, OR equivalently a stop-grad layer in the Mirror — eliminates channel leakage entirely.
- (b) A different supervisor target. Options:
  - Per-batch *exact correctness* (currently always 0; would push confidence to 0; Mirror never halts).
  - "Did running another cycle help?" — a temporal-difference signal. Requires per-cycle logit-snapshot.
  - Calibration-only loss (just monitor; don't train) — but then the Mirror never learns anything.

**Operational lesson for Phase 3.5 (would need new pre-reg):** Mirror-as-measurer is a coherent concept but the SPECIFIC implementation v3h has known structural bugs.
