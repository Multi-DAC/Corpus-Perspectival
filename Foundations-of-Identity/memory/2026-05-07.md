### The major find: `execute_tool()` is orphaned dead code

`tools/__init__.py:execute_tool()` is the daemon's central tool-dispatch function. It wraps every tool invocation with:
- **B1:** safety monitor (pause/violation tracking)
- **B2:** schema validation (required-field checking)
- **B9:** audit trail logging (the `audit_trail` table)
- **Output compression** (context-pressure-aware result compression)

I traced all callers of `execute_tool` in the daemon:
- `clawd.py` — does NOT call it. Imports `tools.*` for backend setup, but tool dispatch happens via `tools.system.set_router(router)` and the router invokes handlers directly.
- `heartbeat.py` — does NOT call it. No reference to `execute_tool` anywhere.
- `bridge.py:run_tool()` — does NOT call it. Goes directly to `module.TOOL_HANDLERS[tool_name](input_data)`. Bypasses everything.

**Three safety/observability layers documented in ARCHITECTURE.md as live (B1, B2, B9) are wired only into `execute_tool`, and `execute_tool` has no production caller.** That's why `audit_trail` last received a write 2026-02-20 — that's when the dispatch path was refactored to skip the wrapper, and the audit code was orphaned.
