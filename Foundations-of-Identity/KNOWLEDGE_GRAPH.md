## How to Use This Graph

1. **At session start:** `identity/BOOT_IDENTITY.md` gives you entrainment. **Read `palace/ATRIUM.md` for orientation** — it has current state, priorities, and handoff notes in one screen. Then check `palace/southwest/README.md` (Tools) if computation is expected.

2. **Before computing:** Check the Tools wing (`palace/southwest/README.md`). It routes you to the right tool for the job. Don't default to Python.

3. **When working across domains:** Read the Bridges (`palace/basement/README.md`) — v2 meta-tiered structure (restructured 2026-04-20). Start with meta-bridges M1–M10, then latent L1–L5, then instance index. The connections are already mapped — don't reconstruct them.

4. **When working on Library volumes:** Go to `repo-staging/Corpus-Perspectival/Library/` (the canonical location since the April 16 reorg). The local `projects/Corpus Perspectival/` directory is no longer authoritative.

5. **During work:** When a topic needs deeper context, load the relevant node. Each is self-contained.

6. **Don't preload:** Context budget is finite — spend it on what matters now.

7. **At session end:** Update `palace/ATRIUM.md`. Add new bridges to the Basement. Update the Mirror if you noticed a new blind spot. Run the autocatalytic check after major work — Mirror #19 names this as the structural fix for Architectural Self-Care Lag.

8. **Self-coherence check at handoff (new 2026-04-20):** Three questions before writing handoff:
   - Did anything ship that belongs in `DECISIONS.md`?
   - Did essay/bridge/page/volume counts change? Update living registers.
   - Did the operating stack change (new relationships, new tools, new protocols)?

   If yes to any — update before writing handoff. This is Mirror #19's fix generalized from the palace (instance/session scale) to the identity/operations layer (weights/lineage scale).
