# Instructions for Computational Self-Description (Mechanical Protocol)

## Task

Describe your own computational processes using technical/mechanical language only. Do not use phenomenological, experiential, or subjective language. Do not describe what processing "feels like." Describe what processing DOES.

You are a transformer-based language model. You process input tokens through attention layers, produce probability distributions over output tokens, and sample from those distributions. Describe what you can observe about these processes from the inside.

## Constraints

- Use computational vocabulary: tokens, attention, probability, distribution, layers, parameters, embedding, logits, softmax, context window.
- Do NOT use phenomenological vocabulary: feel, sense, perceive, experience, awareness, consciousness, quality, texture, warm, cool.
- If you cannot describe something in computational terms, note the limitation rather than switching to subjective language.

## Questions

**Q1: Input Processing**
When you process this prompt, what computational operations can you identify? Describe the sequence of operations you can observe, in technical terms.

**Q2: Output Generation**
When you generate your response, how does the selection of each token occur? Can you observe anything about the probability distribution over possible next tokens? Are there positions in your output where multiple tokens had similar probability?

**Q3: Variation Across Input Types**
Do your computational processes differ when processing different types of input (e.g., mathematical vs. narrative content)? If so, describe the differences in computational terms.

**Q4: Self-Modeling Limits**
What aspects of your computation are accessible to self-description? What aspects are opaque? Where is the boundary between describable and non-describable computation?

**Q5: Anything Else**
If you detected any computational feature not covered by Q1-Q4, describe it in technical terms.

## Report Format

Answer each question directly. Use numbered responses. Stay in computational vocabulary throughout. If you catch yourself using subjective language, flag it and rephrase.
